# Contributing to Odoo 19 Masterclass

First off, thank you for considering contributing to the Odoo 19 Masterclass! It's people like you that make this a world-class resource for the community.

## 🚀 How Can I Contribute?

### Reporting Bugs 🐛
If you find a typo, a broken link, or a technical error in the documentation:
1. Check the [Issues](https://github.com/Kashyap-001/odoo19-docs/issues) to see if it has already been reported.
2. If not, open a new issue. Please provide as much detail as possible, including the page URL and a description of the error.

### Suggesting Improvements 💡
Have an idea for a new lesson or a clearer way to explain a concept?
1. Open an [Issue](https://github.com/Kashyap-001/odoo19-docs/issues) with the tag `enhancement`.
2. Describe the improvement and why it would be valuable for other learners.

### Pull Requests (The Merge Request Flow) 🤝
If you want to contribute code or documentation changes directly:
1. **Fork** the repository to your own GitHub account.
2. **Clone** your fork locally.
3. Create a **Branch** for your changes (`git checkout -b feature/my-improvement`).
4. **Commit** your changes with a clear and concise message.
5. **Push** your branch to your fork.
6. Open a **Pull Request** (Merge Request) from your fork's branch to our `main` branch.

## 🎨 Style Guidelines

### Documentation (Markdown)
- Follow the established architecture patterns.
- Keep the "Senior Insight" and "Senior Checkpoint" format for consistency.
- Use clear, professional, and senior-architect level language.

### Code Examples
- Ensure all Python code follows Odoo 19 / Python 3.12 standards.
- Ensure all OWL/JS code follows modern OWL 2.0 standards.
- Use the `pways_auction` module context wherever possible for consistency.
